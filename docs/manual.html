SMS LocalBlast — User & Developer Manual

Version 1.0

1. Introduction

SMS LocalBlast is a white-label SMS gateway: Excel → Node.js API → mobile gateways → SMS/MMS. You use your own phone plan; no per-message fees.

  • Excel Add-in — Send bulk SMS from spreadsheets
  • API — REST endpoints for auth, queue, devices
  • Android App — Polls API, sends via native SMS
  • iOS Shortcut — Polls API, sends via Shortcuts app
  • Personalization — {{Name}}, {{Balance}}, etc.
  • Multi-device — Load balancing across phones

2. Installation

2.1 Prerequisites

  • Node.js 18+
  • A Supabase project

2.2 Database (Supabase)

  1. Create a project at supabase.com
  2. Run server/sql/001_initial.sql in SQL Editor
  3. Run server/sql/002_claim_next_message.sql
  4. Optional: 005_licenses_and_global_settings.sql, 006_claim_with_device.sql, 007_api_keys.sql

2.3 Web Installer

  1. cd server && npm run bootstrap
  2. Edit .env with Supabase URL and keys (or leave for installer)
  3. npm run dev
  4. Visit http://localhost:3000/install
  5. Complete the 3 steps: Database, Branding, Admin
  6. Restart the server

2.4 Manual .env

SUPABASE_URL=https://xxx.supabase.co
SUPABASE_PUBLISHABLE_KEY=eyJ...
SUPABASE_SERVICE_ROLE_KEY=eyJ...
JWT_SECRET=<openssl rand -hex 32>

Never expose SUPABASE_SERVICE_ROLE_KEY to clients.

3. Excel Add-in

Sideload

  1. Start the server: cd server && npm run dev
  2. Copy excel-addin to a shared folder (e.g. C:\SMS-LocalBlast-Addin)
  3. Excel → Insert → Get Add-ins → My Add-ins → Shared Folder
  4. Select the folder with manifest.xml
  5. Insert → My Add-ins → SMS LocalBlast

Sheet format

Column A (Phone)Column B (Message)Column C (Media URL, optional)
+15551234567Hi {{Name}}!

Use {{Name}}, {{Balance}}, etc. — values come from row data. Demo mode: demo@localblast.com / any password (no real SMS).

4. Android Gateway

Requirements: Android 9+, Android Studio.

  1. Open android/ in Android Studio
  2. Edit app/src/main/res/values/config.xml — set API base URL
  3. Emulator: http://10.0.2.2:3000
  4. Real device (same Wi‑Fi): http://YOUR_PC_IP:3000
  5. Build → Run

Login → Start gateway. App polls every ~15–30s with smart jitter, claims messages, sends via native SMS, reports status.

5. iOS Shortcut

Requirements: iOS 15+, Shortcuts app.

  1. Create a shortcut with actions from ios-shortcut/RECIPE.md
  2. Set API URL and token (from POST /api/auth/login)
  3. Run manually or via Personal Automation (e.g. every 2 min)

Visit /ios-shortcut for setup guide and QR code.

6. API Reference

Auth

POST /api/auth/register
{"email":"user@example.com","password":"secret","terms_accepted":true}

POST /api/auth/login
{"email":"user@example.com","password":"secret"}
→ {"token":"...", "user":{...}}

Messages (Bearer token)

POST /api/messages
{"to_phone":"+15551234567","body":"Hello","media_url":"https://..." (optional)}

POST /api/messages/claim-next
{"device_id":"optional"}
→ {"message":{...} or null}

PATCH /api/messages/:id/status
{"status":"sent"|"delivered"|"failed"}

Devices

POST /api/devices
{"name":"My Phone","platform":"android"}

Admin (admin role)

GET /api/admin/users
GET /api/admin/stats
GET /api/admin/api-keys
POST /api/admin/api-keys
DELETE /api/admin/api-keys/:id
GET/PATCH /api/admin/brand
POST /api/admin/update

7. Admin Dashboard

Visit /admin, sign in with an admin account. Create one: UPDATE users SET role = 'admin' WHERE email = 'you@example.com';

Features: stats, users table, API key manager, brand settings (site name, support email, primary color), server health, update button.

8. Licensing

Optional. Deploy license-server/, run 005_licenses_and_global_settings.sql on its Supabase, add purchase codes. In main server .env: LICENSE_SERVER_URL, PURCHASE_CODE. See license-server/README.md.

9. Troubleshooting

"violates row-level security policy"
Run 003_disable_rls_for_api.sql or use the service_role key.
Excel add-in won't load
Ensure manifest path is trusted; server must be running; CORS allows origin.
Android can't reach API
Use 10.0.2.2:3000 for emulator; use PC's LAN IP for real device; check firewall.
claim_next_message missing
Run 002_claim_next_message.sql in Supabase.
Rate limited (429)
~200 msg/hour per user by default. Adjust RATE_LIMIT_MSG_PER_HOUR.
